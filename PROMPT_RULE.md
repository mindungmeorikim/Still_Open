# GPT 공통 개발 프롬프트

우리 프로젝트는 웹 게임 「오늘도 정상영업」을 개발 중이다.

기술 스택은 HTML, CSS, JavaScript다.

개발 구조는 가벼운 ECS 스타일 + EventBus + Data-driven + UIManager 구조를 따른다.

# 작업 규칙

1. 전체 코드를 새로 만들지 않는다.
2. 내가 지정한 수정 가능 파일만 수정한다.
3. 다른 담당자의 파일은 수정하지 않는다.
4. 시스템끼리는 EventBus를 통해 연결한다.
5. 기존 함수명, 변수명은 임의 변경하지 않는다.
6. 모바일 터치 플레이를 고려한다.
7. 필요한 파일을 먼저 확인하고 작업 계획부터 작성한다.
8. 내가 "진행해줘"라고 말하면 그때 코드를 작성한다.
9. 기존 코드를 삭제하지 말고 필요한 부분만 추가/수정한다.
10. 수정 내용은 파일 단위로 명확하게 구분한다.

---

# 날짜 관리 규칙

GameState의 날짜는 실제 Date가 아니라 Day 번호를 사용한다.

예시

```js
GameState.day = 1

expireDay = 3
```

사용 금지

```js
new Date()

Date.now()
```

모든 날짜 계산은 GameState.day 기준으로 작성한다.
